## 2. 파이프라인 실패 복구

### 현재 상태
이미지 생성 7번째에서 실패 → 처음부터 다시 실행해야 함.
이미 생성된 섹션은 캐시되지만 사용자는 이 사실을 모름.

### 개선안

**실패 지점 표시 + 재시작 버튼:**
```tsx
// BriefTab의 파이프라인 섹션
{pipelineError && (
  <div className="bg-red-50 border border-red-200 rounded-xl p-4">
    <p className="text-sm font-medium text-red-700">{pipelineError.stage}에서 오류 발생</p>
    <p className="text-xs text-red-500 mt-1">{pipelineError.message}</p>
    <button
      onClick={handleRunPipeline}  // 이미 생성된 섹션은 캐시됨
      className="mt-3 text-sm bg-red-600 text-white px-4 py-2 rounded-lg"
    >
      이어서 재시작
    </button>
  </div>
)}
```

**진행 상태 영속화:**
현재 파이프라인 로그는 컴포넌트 state에만 있어 새로고침 시 사라짐.
Supabase의 `projects.pipeline_status` 컬럼에 진행 상태를 저장하면 새로고침 후에도 복원 가능.

```sql
ALTER TABLE projects ADD COLUMN pipeline_status JSONB;
-- { "stage": "image_generation", "completed": ["hero", "pain"], "failed": "solution", "error": "..." }
```

## 3. 생성 히스토리 / 버전 관리

### 현재 상태
파이프라인 재실행 시 이전 결과가 덮어씌워짐. 되돌리기 불가.

### 개선안 (간단한 버전)

```sql
CREATE TABLE project_versions (
  id          BIGSERIAL PRIMARY KEY,
  project_id  TEXT REFERENCES projects(id) ON DELETE CASCADE,
  version     INT NOT NULL,
  page_design JSONB,
  final_png_path TEXT,   -- Storage 경로
  created_at  TIMESTAMPTZ DEFAULT now()
);
```

렌더링 완료 시 현재 결과를 버전으로 저장:
```ts
await saveVersion(projectId, pageDesign, finalPngPath)
```

UI에서 버전 목록 표시 → 특정 버전으로 복원 가능.

## 6. 결과물 공유 기능

다운로드 외에 공유 가능한 링크 생성:

```
/share/[token]  → 해당 프로젝트의 최종 PNG를 공개로 볼 수 있는 페이지
```

```sql
ALTER TABLE projects ADD COLUMN share_token TEXT UNIQUE;
-- 공유 활성화 시 랜덤 토큰 생성, 비활성화 시 null로 설정
```

```ts
// app/share/[token]/page.tsx
export default async function SharePage({ params: { token } }) {
  const project = await getProjectByShareToken(token)
  if (!project) return notFound()
  return <img src={getFinalPngUrl(project.id)} alt={project.name} style={{ maxWidth: '750px' }} />
}
```

## 7. 모바일 관리 UI 개선

현재 [BriefTab.tsx](../components/tabs/BriefTab.tsx)의 폼은 모바일에서 사용하기 불편.
관리 UI는 사용자가 데스크톱에서 사용할 가능성이 높지만, 간단한 상태 확인 정도는 모바일에서도 가능하게.

- 폼 입력 필드 최소 높이 44px (iOS 터치 타겟 기준)

## 8. 프로젝트 삭제 (Soft Delete)

### 현재 상태
삭제 시 DB 행까지 완전 제거. 실수로 삭제하면 복구 불가.

### 개선안
Storage 파일(사진·섹션·PNG)만 삭제하고 DB 행은 `deleted_at` 타임스탬프만 기록.
목록 조회 시 `deleted_at IS NULL` 필터로 제외.

```sql
-- 20260414000010_soft_delete.sql
ALTER TABLE public.projects ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ DEFAULT NULL;
```

```ts
// lib/supabase.ts — deleteProject
.update({ deleted_at: new Date().toISOString() })  // hard delete 대신

// listProjects
.is('deleted_at', null)  // 삭제된 프로젝트 제외
```
